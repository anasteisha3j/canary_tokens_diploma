🌍 Публічна адреса (URL-токени)

Змінна середовища PUBLIC_BASE_URL підставляється в нові URL-токени. Старі токени не оновлюються — після зміни адреси згенеруйте нові.

Приклад: {{ public_base_url }}/{{ canary_link_prefix }}/Ab3xY9mK2nQpL5wR — без UUID і без /api/trigger/. Не ставте api, tokens, static тощо.

Перевірка з іншої мережі або з мобільного інтернету (не Wi‑Fi ПК):

  • Відкрийте в браузері: {{ public_base_url }}/api/ping — очікується JSON {"ok": true}.
  • Якщо не відкривається: ПК не доступний ззовні (CGNAT, немає port forwarding, фаєрвол), або використовується http на порту, який провайдер блокує — тоді потрібен тунель (ngrok, Cloudflare Tunnel) або VPS.
  • З Wi‑Fi для LAN використовуйте http://<локальний IPv4 ПК>:5000 у PUBLIC_BASE_URL і знову згенеруйте URL-токени.
  • Якщо ping працює, а trigger ні: токен міг уже спрацювати — створіть новий або подивіться відповідь JSON (може бути already_triggered).
